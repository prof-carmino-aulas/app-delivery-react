# Delivery PWA

O **Delivery PWA** é um aplicativo Web/PWA de delivery desenvolvido com **React**, **TypeScript** e **Vite**. O projeto foi concebido como uma base didática para demonstrar a construção gradual de uma aplicação moderna, organizada em módulos e orientada à separação de responsabilidades. Mais do que apresentar uma interface funcional, o projeto busca evidenciar como decisões arquiteturais simples, quando bem estruturadas desde o início, favorecem a evolução, a manutenção e a compreensão do código.

A proposta do aplicativo está baseada em um fluxo **guest-first**, no qual o usuário pode navegar pela aplicação, consultar restaurantes, visualizar informações, acessar o carrinho e avançar na experiência de compra antes de realizar login ou cadastro. Essa decisão é importante porque aproxima o sistema de uma experiência real de uso, reduzindo barreiras iniciais e permitindo que a autenticação seja solicitada apenas quando ela se torna necessária para a continuidade do processo.

A estrutura inicial da aplicação foi construída com foco na clareza do fluxo de execução. O arquivo `main.tsx` representa o ponto de entrada do projeto. Nele, o React é conectado ao elemento `root` presente no HTML, a raiz da aplicação é criada com `createRoot`, a interface é envolvida por `StrictMode` e o sistema de rotas é ativado por meio do `RouterProvider`. Esse arquivo concentra a inicialização da aplicação e estabelece o vínculo entre a estrutura HTML e a aplicação React.

O roteamento da aplicação é definido em `src/app/router.tsx`, utilizando o React Router. Esse arquivo centraliza as rotas principais do aplicativo, como a página inicial, a busca, a página de restaurante, o carrinho, o checkout e a tela de acompanhamento do pedido. A rota `/restaurante/:restaurantId` utiliza um parâmetro dinâmico, permitindo que diferentes restaurantes sejam acessados a partir de um identificador presente na URL. Essa organização torna o fluxo de navegação mais explícito e facilita a expansão futura do projeto.

O layout principal está concentrado no componente `AppLayout`, localizado em `src/layouts/AppLayout.tsx`. Esse componente organiza a estrutura visual comum a todas as páginas, incluindo o cabeçalho, a área principal de conteúdo e a navegação inferior. O componente `<Outlet />`, fornecido pelo React Router, define o ponto exato em que as páginas filhas serão renderizadas. Dessa forma, o layout permanece fixo, enquanto o conteúdo central muda conforme a rota acessada pelo usuário.

A navegação principal utiliza o componente `NavLink`, também fornecido pelo React Router. O uso de `NavLink` é adequado porque permite identificar quando uma rota está ativa, possibilitando aplicar estilos específicos ao item de menu correspondente. No projeto, a navegação inferior contém links para Home, Busca, Carrinho, Checkout e Status do pedido. Essa escolha reforça a clareza da navegação e melhora a percepção do usuário sobre sua posição dentro da aplicação.

O estado global do carrinho é controlado com a biblioteca **Zustand**, no arquivo `src/core/stores/app.store.ts`. A store mantém a lista de itens do carrinho, a quantidade total de itens e as funções responsáveis por adicionar produtos e limpar o carrinho. O uso do Zustand permite compartilhar estado entre diferentes componentes sem recorrer a estruturas mais complexas ou a repasses excessivos de propriedades. Essa decisão mantém o código simples, direto e adequado ao porte inicial do projeto.

A função `addCartItem` verifica se um item já existe no carrinho. Quando o item já está presente, sua quantidade é incrementada. Quando o item ainda não existe, ele é adicionado com quantidade inicial igual a um. O total de itens é recalculado com `reduce`, garantindo que a informação exibida no badge do carrinho represente sempre a soma real das quantidades. Esse trecho ilustra uma prática essencial em React: atualizar o estado de forma imutável, criando novas estruturas em vez de alterar diretamente os dados existentes.

Os dados utilizados pela interface são simulados e ficam organizados em `src/core/mocks/delivery.mock.ts`. Esse arquivo define tipos TypeScript para categorias e restaurantes, além de armazenar listas mockadas que alimentam a página inicial e a página de detalhes do restaurante. Essa estratégia é importante porque permite desenvolver e validar a interface antes da integração com uma API real. O uso de mocks também favorece a explicação didática do projeto, pois separa claramente a interface da origem dos dados.

A página inicial, representada por `HomePage`, apresenta a chamada principal do aplicativo, as categorias e os restaurantes disponíveis. Ela utiliza renderização de listas com `map`, permitindo transformar arrays de dados em elementos visuais. Cada item renderizado recebe uma propriedade `key`, necessária para que o React identifique corretamente os elementos durante as atualizações da interface. Essa página demonstra como o React permite construir interfaces orientadas a dados.

A página de restaurante, representada por `RestaurantPage`, utiliza o hook `useParams` para recuperar o identificador do restaurante presente na URL. Com esse identificador, o código busca o restaurante correspondente na lista de dados simulados. Caso o restaurante não seja encontrado, a aplicação exibe uma mensagem apropriada. Esse cuidado evita que a interface tente renderizar dados inexistentes e demonstra a importância de tratar estados inválidos ou inesperados.

A página do carrinho, representada por `CartPage`, acessa o estado global da aplicação para verificar a quantidade de itens adicionados. A interface utiliza renderização condicional para apresentar mensagens diferentes quando o carrinho está vazio ou quando possui itens. Esse comportamento evidencia uma característica central do React: a interface é uma representação do estado atual da aplicação.

A página de checkout representa a etapa de finalização do pedido. Neste módulo inicial, ela ainda funciona como uma estrutura visual para evolução futura. O objetivo é preservar o fluxo guest-first, permitindo que a autenticação ou identificação do usuário seja solicitada apenas em uma etapa mais avançada do processo. Essa escolha mantém a experiência inicial mais fluida e reduz a interrupção prematura da jornada do usuário.

A página de acompanhamento, representada por `TrackingPage`, exibe as etapas do pedido. Ela utiliza uma lista de etapas e aplica classes condicionais para diferenciar visualmente etapas concluídas e pendentes. Esse recurso demonstra como a interface pode representar o progresso de um processo de forma clara, oferecendo feedback visual ao usuário.

O projeto utiliza a biblioteca **React Icons** para inserir ícones na interface. Os ícones são tratados como componentes React e podem ser usados diretamente no JSX. No layout principal, por exemplo, são utilizados ícones para representar Home, Busca, Carrinho, Checkout, Status e a identidade visual do aplicativo. Essa abordagem melhora a expressividade visual da interface sem comprometer a organização do código.

A aplicação também considera aspectos básicos de acessibilidade. A navegação principal recebe o atributo `aria-label`, permitindo que tecnologias assistivas identifiquem sua função. Os ícones decorativos utilizam `aria-hidden="true"`, evitando que leitores de tela anunciem elementos que não acrescentam informação textual relevante. Esses detalhes são importantes porque mostram que a qualidade de uma interface não depende apenas da aparência, mas também da sua estrutura semântica.

A estilização global da aplicação está no arquivo `src/index.css`. Esse arquivo define variáveis CSS para cores, tamanhos, espaçamentos, sombras e bordas. O uso de variáveis permite manter consistência visual e facilita futuras alterações de tema. A identidade visual utiliza como referência o verde escuro `#215A37` e o azul escuro `#14387F`, criando uma base cromática sóbria, adequada para uma aplicação institucional e didática.

O CSS também implementa responsividade, permitindo que a interface se adapte a diferentes tamanhos de tela. A navegação inferior, por exemplo, pode assumir comportamento diferente em dispositivos móveis e em telas maiores. Esse cuidado é relevante porque uma aplicação Web/PWA precisa funcionar adequadamente em contextos variados de uso, incluindo celulares, tablets e desktops.

Os testes automatizados são construídos com **Vitest** e **Testing Library**. O arquivo `src/app/App.test.tsx` verifica se a página inicial é renderizada corretamente e se a navegação principal está presente. A Testing Library prioriza consultas por papel semântico, como `getByRole`, aproximando os testes da forma como o usuário e as tecnologias assistivas percebem a interface. Essa abordagem torna os testes mais significativos do ponto de vista da experiência real de uso.

Para executar corretamente os testes, o Vite deve estar configurado com ambiente `jsdom`, permitindo simular recursos de navegador durante a execução dos testes. Também é necessário importar `@testing-library/jest-dom` em um arquivo de setup, para habilitar matchers como `toBeInTheDocument`. Essa configuração integra o ambiente de desenvolvimento ao ciclo de validação da interface.

Para instalar as dependências do projeto, utiliza-se o comando `npm install`. Para executar a aplicação em ambiente de desenvolvimento, utiliza-se `npm run dev`. Para executar os testes, utiliza-se `npm run test`. Para gerar uma versão de produção, utiliza-se `npm run build`. Esses comandos representam o ciclo básico de trabalho em um projeto React com Vite.

Este projeto trabalha conceitos fundamentais de React, como componentes funcionais, JSX, roteamento, rotas aninhadas, parâmetros de URL, renderização condicional, renderização de listas, estado global, hooks, tipagem com TypeScript, organização modular, responsividade e testes de interface. Cada conceito aparece associado a uma necessidade concreta do aplicativo, o que favorece a aprendizagem progressiva e contextualizada.

Em síntese, o **Delivery PWA** representa a base inicial de uma aplicação React moderna. O projeto ainda não implementa toda a regra de negócio de um aplicativo de delivery completo, mas estabelece uma fundação consistente para evolução futura. A partir dessa estrutura, é possível avançar para busca real, cardápio por restaurante, carrinho funcional, cálculo de totais, checkout, autenticação, persistência local, integração com API e comportamento completo de PWA.

Do ponto de vista didático, o valor do projeto está justamente na sua organização incremental. Ele não tenta resolver todos os problemas de uma única vez. Em vez disso, apresenta uma estrutura inicial clara, compreensível e evolutiva. Essa é uma decisão importante no ensino de desenvolvimento de software, pois permite que o estudante compreenda primeiro a arquitetura da aplicação para, em seguida, aprofundar regras de negócio, integração com serviços, persistência e experiência do usuário.
